# Как подключить AI-чат (DeepSeek → позже OpenAI)

Порядок действий для менеджера. После шагов ниже можно переписываться с чатом в приложении.

## 1. Завести аккаунт DeepSeek

1. Откройте https://platform.deepseek.com/
2. Зарегистрируйтесь / войдите.
3. Пополните баланс (для тестов обычно хватает небольшой суммы).
4. Раздел **API Keys** → создайте ключ → скопируйте сразу (потом не покажут).

Ключ выглядит примерно так: `sk-...`

**Не коммитьте ключ в git и не присылайте его в публичные чаты.**  
Достаточно сохранить у себя и передать разработчику через безопасный канал (личный мессенджер / менеджер паролей), либо вставить только локально у себя при запуске.

## 2. Что нужно от вас (чек-лист)

| # | Что | Обязательно? | Куда |
|---|-----|--------------|------|
| 1 | API-ключ DeepSeek (`sk-...`) | Да | локальный запуск / CI secret |
| 2 | Подтверждение: тесты на DeepSeek OK | Да | сообщение в чат разработки |
| 3 | Позже: ключ OpenAI (или другого провайдера) | Нет, этап 2 | когда решим сменить модель |
| 4 | Правки текста системного промпта | По желанию | файл `prompts/ai-chat-system.md` |

Токен сессии AutoDoctor (гостевой) для AI **не нужен** — чат на этапе теста ходит в DeepSeek напрямую с API-ключом.

## 3. Как запустить приложение с ключом

В терминале из папки `apps/mobile`:

**Windows (PowerShell):**

```powershell
puro flutter run --dart-define=DEEPSEEK_API_KEY=sk-ВАШ_КЛЮЧ
```

**Опционально** (если DeepSeek сменит URL/модель):

```powershell
puro flutter run `
  --dart-define=DEEPSEEK_API_KEY=sk-ВАШ_КЛЮЧ `
  --dart-define=DEEPSEEK_BASE_URL=https://api.deepseek.com `
  --dart-define=DEEPSEEK_MODEL=deepseek-chat
```

Без `DEEPSEEK_API_KEY` чат откроется, но ответит, что ключ не настроен.

## 4. Как пользоваться в приложении

1. Вкладка **AI** → список тем (история чатов).
2. **Новый чат** → экран переписки (`/ai/chat/...`).
3. Либо откройте старую тему из списка.
4. Пишите внизу, ответы появляются сверху в ленте.

## 5. Файл промпта

Канонический текст для агента: [`prompts/ai-chat-system.md`](ai-chat-system.md).

Туда можно вносить:
- тон и правила ответов;
- границы ответственности;
- термины AutoDoctor (План / Состояние / Журнал);
- сезонные или продуктовые акценты.

После правок файла пересоберите приложение (или скажите разработчику синхронизировать встроенную копию в коде).

## 6. Переход на OpenAI позже

1. Ключ на https://platform.openai.com/api-keys  
2. Сообщите разработчику: провайдер `openai`, модель (например `gpt-4o-mini`).  
3. В коде уже заложен OpenAI-совместимый формат (`/v1/chat/completions`) — смена в основном в `dart-define` / конфиге, не в UX.

## 7. Безопасность (важно)

- Ключ LLM **не кладём** в репозиторий, README с реальным ключом, скриншоты.
- Для продакшена ключ лучше держать на сервере Laravel (прокси), а не в мобильном приложении.
- Сейчас прямой вызов из приложения — **только для локальных тестов MVP**.
